Edge q-labelling¶

  • ND snaps are used to label PRN0 edges with q, a real value signifying a temporal order on edges.
    • q of an edge records the q (of the snapshot) where the edge first appears in an ND run.
    • Edge q-labels are distinct from edge contact probabilities, which are a function of Q, and is cumulative.
  • Use the edge q-labels to order edges; such an ordering makes the characteristic edge formation order for an ND variant.

  • How effective are these characteristic edge orderings of ND variants in identifying TSE PRN0 edges (Paci dataset)?
    • Are PRN0 edges which are formed earlier (have smaller avg_q), more likely to be in TSE PRNs? This is a desirable outcome. Compare AUCs of ND variants.

def plot_Paci_ROCs(tp_col:str, plt_title: str):
    fig, axes = plt.subplots(3, 4, figsize=(12, 9), constrained_layout=True)
    axes[2][2].remove(); axes[2][3].remove()
    plt.suptitle(plt_title, fontsize=14)
    AUCs = {} # keyed by pid
    for i, pid in enumerate(PIDs):
        r = i//4; c = i%4; ax = axes[r][c]
        fn = os.path.join(dirname, pid + "_base_edge_features.out")
        de = pd.read_table(fn, sep=' ')
        TP_labels = np.where(de[tp_col].values >= 0.5, 1, 0) # TP in eid order
        scores = []
        for j, ndv in enumerate(NDVs):
            fn = os.path.join(dirn, pid + "_edge_qlabels_" + ndv + ".out")
            dq = pd.read_table(fn, sep=' ', skiprows=1, header=None).dropna(axis=1)
            dq.columns = ['eid', 'x', 'y', 'sc0', 'avg_q', 'mode_q', 
                          'q1', 'q2', 'q3', 'q4', 'q5', 'q6', 'q7', 'q8', 'q9', 'q10'] 
            df = pd.DataFrame({'avg_q': dq.avg_q, 'TP': TP_labels})
            df = df.sample(frac=1.0)
            df.sort_values(by='avg_q', inplace=True)
            y_true = df.TP.values; y_score = np.flip(df.avg_q.values)        
            fpr, tpr, _ = roc_curve(y_true, y_score)    
            auc = ras(y_true, y_score); scores.append(auc)
            ax.plot(fpr, tpr, label=ndv + ' ' + str(np.round(auc, 2)))
        AUCs[pid] = scores
        ax.set_title(pid)
        ax.plot([0.0, 1.0], [0.0, 1.0], '--', color='black')    
        if 0==r: ax.set_xlabel("FPR", fontsize=14)
        if 0==c: ax.set_ylabel("TPR", fontsize=14)    
        ax.legend(title='AUC'); ax.grid()
    aucs_df = pd.DataFrame.from_dict(AUCs, columns=NDVs, orient='index')        
    return aucs_df

aucs_df = plot_Paci_ROCs("tse_prn_prob", "Identifying TSE PRN0 edges with avg_q.")
  • First sanity check: 'r' ROCs (green) hug the diagonal (y=x) line, the 'y' ROCs (purple) lie below (to the right of) this diagonal, and the ROCs of the biased ND variants ('a', 'c', 'x') are above this diagonal.
  • Can conclude that the characteristic edge orderings produced by the biased ND variants are able to identify PRN0 edges that are present in TSE PRNs, over those which are absent. They produce "sensible" edge orderings as far as TSE PRN edges go.
  • No noteworthy difference between the biased ND variants (mean acx AUCs =0.80) on this task.
    • Although at very small FPR, 'x' and 'c' score higher TPR than 'a' for several pids.
  • Similar qualitative results for SCN0 edges in TSE PRNs (mean acx AUCs =0.70).
